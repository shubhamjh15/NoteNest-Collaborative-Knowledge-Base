## üõ†Ô∏è Contribution Workflow

All contributions must follow this workflow:

1. **Choose an Issue**
   - Look for OSQ-labeled issues
   - Beginners should start with `good first issue`

2. **Fork the Repository**
   - Create a fork under your GitHub account

3. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
4.Make Changes

  -Follow project coding standards
  
  -Write clear commit messages

5.Open a Pull Request (PR)

  -Clearly describe what you changed
  
  -Link the issue number
  
  -Add screenshots if applicable

6.Address Review Feedback
  
  -Respond respectfully

  -Make requested improvements

### Evaluation Criteria (OSQ)

Contributions are evaluated based on:

| Criteria      | Description                        |
| ------------- | ---------------------------------- |
| Quality       | Clean, readable, maintainable work |
| Impact        | How useful the contribution is     |
| Consistency   | Regular, genuine participation     |
| Communication | PR descriptions & discussions      |
| Learning      | Growth and effort shown            |

‚ö†Ô∏è Quantity alone will not guarantee a high score.
